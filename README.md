# 应急物资轮换和借用系统

一个完整的应急物资管理系统，支持物资批次管理、借用审批流程、库存轮换、审计追踪等功能。

## 技术栈

- **后端**：Python 3.x + Flask + SQLAlchemy + SQLite
- **前端**：Vue 3 + Vite + Element Plus + Pinia
- **认证**：JWT Token
- **数据存储**：SQLite 本地数据库（无需额外安装）

## 快速启动

### 1. 启动后端服务

```bash
# 进入后端目录
cd backend

# 安装依赖（首次运行）
pip install -r requirements.txt

# 启动服务
python app.py
```

后端服务将在 `http://localhost:5000` 启动，首次启动会自动创建数据库和初始化用户账号。

### 2. 启动前端服务

```bash
# 进入前端目录
cd frontend

# 安装依赖（首次运行）
npm install

# 启动开发服务
npm run dev
```

前端服务将在 `http://localhost:5173` 启动。

### 3. 访问系统

打开浏览器访问 `http://localhost:5173`

## 数据模型

### 物资批次 (MaterialBatch)
- `batch_no`：批次号（唯一）
- `material_name`：物资名称
- `specification`：规格型号
- `total_quantity`：总数量
- `available_quantity`：可用数量
- `unit`：单位
- `production_date`：生产日期
- `expiry_date`：有效期至
- `warehouse_location`：库位
- `status`：状态（normal/rotated）
- `supplier`：供应商
- `remark`：备注

### 借用单 (BorrowOrder)
- `order_no`：单据号
- `status`：状态（pending_approval/approved/rejected/received/returned/damaged）
- `purpose`：借用用途
- 审批人、领用人、归还人及时间

### 审计日志 (AuditLog)
记录每一次状态变化，包含：
- 操作类型、操作人、操作时间
- 原因/备注（必填）
- 旧状态 → 新状态
- 操作详情

## 三条验证链路

### 链路一：主流程验证（创建批次 → 借出 → 归还）

**步骤 1：创建物资批次**
1. 登录 admin / admin123
2. 进入「物资批次管理」→ 点击「新增批次」
3. 填写信息：
   - 批次号：MASK-2024-001
   - 物资名称：N95口罩
   - 数量：100 盒
   - 生产日期：2024-01-01
   - 有效期至：2026-12-31
   - 库位：A-01-01
   - 入库原因：应急物资采购入库
4. 确认创建成功，查看批次列表中新增的记录

**步骤 2：发起借用申请**
1. 退出登录，登录 duty / duty123
2. 进入「借用单据管理」→ 点击「申请借用」
3. 填写信息：
   - 借用用途：应急演练使用
   - 添加物资：选择 MASK-2024-001，数量 10 盒
4. 提交申请，单据状态变为「待审批」

**步骤 3：审批借用单**
1. 退出登录，登录 admin / admin123
2. 进入「借用单据管理」，找到刚才的申请单
3. 点击「审批」→ 填写审批意见「同意」→ 点击「通过」
4. 单据状态变为「待领用」

**步骤 4：领用物资**
1. 登录 duty / duty123（或 admin）
2. 找到状态为「待领用」的单据，点击「领用」
3. 填写领用备注「现场领用确认」→ 确认领用
4. 单据状态变为「使用中」
5. 查看「物资批次管理」，MASK-2024-001 的可用数量变为 90 盒

**步骤 5：归还物资**
1. 找到状态为「使用中」的单据，点击「归还」
2. 填写归还数量：10 盒，归还备注「使用完毕归还」
3. 确认归还，单据状态变为「已归还」
4. 查看「物资批次管理」，MASK-2024-001 的可用数量恢复为 100 盒

**步骤 6：查看审计历史**
1. 进入「借用单据管理」→ 点击单据「详情」
2. 查看完整的状态流转记录和审计日志
3. 进入「操作审计日志」，查看全系统操作记录

---

### 链路二：异常场景验证（库存不足、提前归还、过期借用）

**场景 1：借用数量超过库存**
1. 确认某批次可用库存（如 90 盒）
2. 发起借用申请，选择该批次，填写数量 100
3. 系统提示：「批次 XXX 库存不足，可用 90盒，申请 100盒」
4. 申请被拒绝，库存保持不变

**场景 2：未领用就归还**
1. 找到一个状态为「待审批」或「待领用」的单据
2. 操作列中不会显示「归还」按钮
3. 直接调用 API 会返回：「当前状态为XXX，无法归还」
4. 库存保持不变

**场景 3：借用已过期批次**
1. admin 创建一个批次，设置有效期为过去的日期（如 2023-01-01）
2. duty 发起借用申请，在物资选择列表中看不到该过期批次
3. 查看该批次详情，状态显示「已过期」
4. 直接调用 API 会返回：「批次 XXX 已过期，不能借用」

**场景 4：报损处理**
1. 领用一批物资（状态变为「使用中」）
2. 点击「报损」，填写报损数量和原因
3. 确认后，单据状态更新，已报损数量增加
4. 物资库存不恢复（因为已报损）

---

### 链路三：数据持久化与导入导出验证

**场景 1：重启服务数据不丢失**
1. 完成链路一的操作，确保已有批次和单据
2. 关闭后端服务（Ctrl+C）
3. 关闭前端服务
4. 重新启动后端：`python app.py`
5. 重新启动前端：`npm run dev`
6. 登录系统，验证：
   - 物资批次列表数据完整，库存数量正确
   - 借用单据状态保持不变
   - 操作审计日志完整保留

**场景 2：CSV 导入（重复批次拒绝）**
1. 进入「物资批次管理」→ 点击「导入批次」
2. 选择 `backend/import_template.csv` 模板文件
3. 填写导入原因「初始化数据导入」
4. 首次导入成功，显示成功 3 条
5. 再次导入同一个文件
6. 系统提示：「导入成功：0 条，跳过：3 条」
7. 查看详情，显示「批次号 XXX 已存在，跳过」
8. 验证数据库中没有重复数据

**场景 3：导出与还原**
1. 进入「物资批次管理」→ 点击「导出批次」
2. 下载 Excel 文件，打开查看关键字段：
   - 批次号、物资名称、规格型号
   - 总数量、可用数量、单位
   - 生产日期、有效期至、库位、状态、供应商
3. 将导出的文件另存为新文件
4. 修改部分字段后重新导入（修改批次号避免重复）
5. 验证新批次数据正确，关键字段完整还原

**场景 4：批次轮换**
1. 选择一个正常批次，点击「轮换」
2. 填写新批次号、新的生产日期和有效期
3. 填写轮换原因「批次到期轮换」
4. 确认轮换后：
   - 旧批次状态变为「已轮换」，可用数量清零
   - 新批次继承旧批次的库存数量，状态为正常
   - 审计日志记录轮换出库和入库两条记录
5. 验证旧批次无法再被借用

## 业务规则总结

1. **库存变更时机**：仅在「领用出库」时扣减库存，「归还入库」时增加库存
2. **状态流转控制**：
   - 待审批 → （审批通过/拒绝）→ 待领用/已拒绝
   - 待领用 → （领用）→ 使用中
   - 使用中 → （归还/报损）→ 已归还/已报损
3. **备注强制要求**：所有状态变更操作必须填写原因/备注
4. **过期批次隔离**：已过期批次在借用选择列表中自动过滤
5. **并发安全**：每次操作都在事务中执行，失败自动回滚，保证库存一致性
6. **审计追踪**：所有操作都记录日志，可追溯到操作人、时间、原因

## 常见问题

**Q: 后端启动报错缺少依赖？**
A: 确保已执行 `pip install -r requirements.txt`

**Q: 前端启动报错找不到模块？**
A: 确保已执行 `npm install`

**Q: 数据库想要重置？**
A: 删除 `backend/data.db` 文件，重新启动后端会自动初始化

**Q: 如何修改端口？**
A: 后端修改 `app.py` 最后一行的 port，前端修改 `vite.config.js` 中的 server.port
